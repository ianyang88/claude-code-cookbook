# 上下文引用与 IDE 集成

## 一句话介绍

@ 文件引用和 IDE 插件联动，高效传递上下文

## 适用场景

- 需要引用多个文件到对话
- 想在 IDE 中可视化选择文件
- 减少复制粘贴代码

## @ 文件选择机制

> CLI 模式下最高效的功能之一，无需复制粘贴代码

### 基本用法

在输入提示词时，输入 `@` 符号：

```
@filename.py    # 引用具体文件内容
@folder/src     # 引用整个文件夹的代码结构
@git-log        # 引用最近的提交记录
```

### 实战示例

```
请分析 @main.java 和 @utils/helper.java 的逻辑，
并在 @tests 目录下生成对应的单元测试文件。
```

## IDE 插件联动

> 将 CLI 的强大能力嵌入开发者最熟悉的编辑器中

### 支持的 IDE

- VS Code
- IntelliJ IDEA

### 安装步骤

1. 打开 VS Code 或 IntelliJ IDEA 的 **插件市场** (Extensions/Plugins)
2. 搜索 `Claude` 或 `Anthropic` 官方/社区插件并安装
3. 输入 `/ide` 联动 IDE 插件

### 核心功能：文件勾选联动

**操作方式**： 在 IDE 侧边栏直接勾选多个文件（支持 Ctrl/Cmd 多选）

**联动机制**： 勾选的文件自动作为上下文注入到 CLI 对话框中

**优势**：
- 极大减少路径输入错误
- 直观管理上下文范围
- 实现"所见即所得"的编码辅助

### IDE vs CLI 对比

| 方式 | 优势 |
|------|------|
| CLI `@` 引用 | 快速、精准、支持文件夹 |
| IDE 勾选 | 可视化、多选方便、所见即所得 |
