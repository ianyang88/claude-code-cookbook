# 核心心法：心态转变

## 一句话介绍

不要把 Claude Code 当成聊天机器人，而要把它当成你的"代理式编码者" (Agentic Coder)

## 适用场景

- 刚开始使用 Claude Code，想充分发挥潜力
- 还在一行行给指令，感觉效率不高
- 想从"提示词工程师"转型为"管理者"

## 核心理念

**从"写代码的工人"到"管理者"的转变**

- 你不再是写代码的工人
- 你是一个管理者
- 你的任务是设定高阶目标、提供上下文
- 最终验证用户体验

## 价值认知

| 心态 | 价值发挥 |
|------|----------|
| 当成 Prompt 工具 | 仅发挥 10% 的价值 |
| 当成代理式编码者 | 发挥 100% 的潜力 |

## 工作方式对比

| 维度 | 过去（提示词工程师） | 现在（管理者/代理模式） |
|------|----------------------|------------------------|
| 交互方式 | 一行行给指令，盯着干活 | 设定高阶目标，提供背景 |
| 关注点 | 抠实现细节，逐行审代码 | 验证用户体验，把控全局 |
| 角色定义 | 微观管理者 | 团队负责人 (Lead) |
| 核心工具 | 复杂的 Prompt | claude.md + 代理流 |

## 关键金句

> 放弃逐行审查代码的旧习惯，在更高的抽象层次验证输出。
> 你未来的价值不再是写出"美的代码"，而是控局。

## 实践建议

1. **设定高阶目标** — 告诉 Claude 你要达成什么，而不是怎么做
2. **提供完整上下文** — 让 Claude 理解项目背景
3. **验证最终结果** — 关注用户体验，而非代码细节
4. **信任代理** — 让 Claude 自主完成中间步骤
