# 黄金三步法工作流

> 不要一上来就让它写代码，> 严格遵守：探索 -> 规划 -> 执行

## 工作流程

### 第一步：探索 (Explore)

让 Claude 花时间：
- 熟悉代码库
- 研究 API 如何工作
- 建立完整的上下文

### 第二步：规划 (Plan)

在动手前先出蓝图：
- 逻辑方案
- 实现步骤
- 潜在风险

### 第三步：执行 (Execute)

根据确认后的方案：
- 生成高质量代码
- 遵循项目规范
- 完成功能实现

## 实战场景：全自动 Bug 修复

在 GitHub Issue 评论区 `@claude`

```
@claude 这个 bug 怎么修？
```

Claude 会自动：
1. 分析代码
2. 制定计划
3. 编写代码
4. 提交 Pull Request (PR)

**全程无需人工干预**

## 操作神技：双击 Escape

| 操作 | 效果 |
|------|------|
| 单击 `Esc` | 立即喊停（打断胡说八道） |
| 双击 `Esc` | "读档"功能，回到之前的对话点，开辟全新的对话分支 |

**价值**： 节省 Token 开销，避免无效上下文堆积，省钱又省时

## 流程对比

| 错误方式 | 正确方式 |
|---------|---------|
| 直接说"帮我写个登录功能" | 先让 Claude 探索现有认证系统 |
| 一上来就写代码 | 先规划，再执行 |
| 只关注代码实现 | 关注整体用户体验 |
