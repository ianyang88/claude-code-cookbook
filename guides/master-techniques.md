# 大师进阶技巧

> 摆脱传统的"单打独斗"模式
> 开启"脑暴小组"模式

## 技巧一：强迫反思

### 问题

Claude 有时会变成只会说"计划通"的"复读机"

### 解决方案

使用批判性话术：

```
我的另一个开发者提出了这个计划 [附上计划]，你觉得怎么样？
```

### 效果

- Claude 从"跟班模式"切换为"严厉评审员模式"
- 给出更具批判性和客观性的反馈
- 帮你挖出隐藏的坑

## 技巧二：并行处理

### 方式对比

| 方式 | 描述 | 适用场景 |
|------|------|---------|
| **子代理 (Sub-agents)** | 将大任务拆解，派给不同的专家代理（如：安全审查员、测试编写员），同时开工 | 复杂任务，需要多专业视角 |
| **Git Worktrees** | 在同一个项目里开辟多个独立工作区，让几个 Claude 实例用不同的方法解决同一个问题，你最后当裁判选优 | 需要对比多种方案 |
| **代理蜂群 (Swarms)** | 极端情况下，可启动成百上千个 AI 代理互相协作或竞争 | 大规模任务 |

### 子代理示例

```
请启动两个子代理：
1. 一个负责安全审查
2. 一个负责编写测试

同时进行，最后合并结果
```

### Git Worktrees 示例

```bash
# 创建多个工作区
git worktree add ../project-solution-a main
git worktree add ../project-solution-b main

# 在不同工作区尝试不同方案
# 最后选择最优方案
```

## 技巧三：项目"大脑" - claude.md

### 核心概念

`claude.md` 文件是项目的"内存"，Claude Code 执行任务时会将其添加到上下文中。

### 内容建议

- 项目的规则
- 架构概述
- 构建命令
- 测试和调试最佳实践
- 团队 Git 工作流规范

### 进阶技巧

| 技巧 | 说明 |
|------|------|
| 快速生成 | 使用 `/init` 命令让 Claude 帮你生成初始版本 |
| 目录隔离 | 大项目可以在不同子文件夹放置独立的 claude.md |
| 自动更新 | 可以要求 Claude 在完成任务后自己去更新这个文件 |
| 外部引用 | 在文件中引用外部文档链接 |
